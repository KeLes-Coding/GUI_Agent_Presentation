研究背景：GUI智能体的困境

评估的模糊性

现有基准（如WebArena）仅依赖最终状态判定。长链路任务中，中间步骤的错误难以被精确衡量，导致“结果正确但过程错误”的假阳性。

规划-执行鸿沟

高层语义指令（如“订一张去北京的票”）与底层精准操作（点击坐标 (x,y)）之间存在巨大语义落差，导致模型幻觉。

稀疏奖励难题

在数十步的长序列操作中，只有最后一步成功才有奖励。一旦失败，传统RL算法（如GRPO）面临严重的“零梯度”问题。

基准构建：NaturalGAIA

Figure 1: Causal Pathway 示意图

核心理念：因果路径

  • 原子任务解构
    将复杂任务拆解为具有严格逻辑依赖的“原子任务链”。
  • 路径崩塌 (Pathway Collapse)
    任何中间节点的信息获取失败，都会切断因果链，判定为无效。
  • 评估范式转移
    从“最终结果导向”转移到“可验证的信息状态流”，消除侥幸成功。

系统架构：LightManus

Task Parser (大脑)

负责语义理解与长时序记忆管理。将用户指令分解为解耦的“原子任务序列”。

Workflow Manager (中枢)

跨平台动态调度层。根据任务类型，将指令分发至 Android (Jarvis) 或 PC 端执行。

Execution Unit (手脚)

在单应用环境下闭环执行具体指令。确保“搜索”、“点击”、“输入”等操作精准落地。

Jarvis 设计：双流感知模块

  • 视觉流 (Visual Stream)

    利用 VLM (如 Qwen2.5-VL) 实时分析屏幕截图，理解图标语义与布局结构。

  • 结构流 (Structural Stream)

    解析 UI Automator XML，剔除视口外无效节点，降低上下文噪音。

  • 创新点：UID 映射

    将复杂的 (x,y) 回归问题转化为基于唯一 ID (uid) 的分类问题，显著提升点击精度。

Jarvis 设计：语义动作与闭环

语义动作模块

摒弃不稳定坐标，采用 tap(uid), input(uid)。

引入物理原语 swipe(dir, mag) 模拟人类直觉操作。

O-T-A 执行闭环

Observe: 获取截图与简化XML。

Think: 输出 JSON {"thought": "...", "action": "..."}。

Act: ADB执行 + 自动重试机制。

核心算法：CCAPO

Counterfactual & Causal Advantage Policy Optimization

LCCAPO = (1-λ) · Lonline + λ · Lbuffer

混合优势函数

宏观防作弊 + 微观因果感知

STDB 轨迹库

Top-K 经验存储与动态更新

混合计算组

解决长序列任务的零梯度问题

混合优势函数：$A_{\text{final}}$

Afinal (at) = Atraj (τ) + ω · Astep (at)

结合宏观轨迹结果与微观步骤贡献，平衡长期目标与短期反馈。

宏观优势 ($A_{\text{traj}}$)

防作弊 (Anti-Shortcut): 若步数过少判定为侥幸，强制归零。

有效奖励: 仅奖励真实有效的成功轨迹。

微观优势 ($A_{\text{step}}$)

引入因果感知，量化单步动作对最终成功的边际贡献。

微观信用分配 (Causal Credit Assignment)

重要性分数 $I_{\text{action}}$

必要性 × 效用性

量化每个动作不仅是"正确的"，而且是"必要的"。

动态权重衰减

随着训练成功率 ($\lambda_{SR}$) 的提升，逐渐降低微观奖励权重。

从依赖“手把手教”过渡到“结果导向”。

动态经验回放 (STDB) 与锚点匹配

从失败中学习

传统方法:

失败轨迹通常被“一刀切”地惩罚，浪费了其中可能存在的正确片段。

CCAPO 方法:

将失败轨迹步骤与 STDB 中的 锚点 (高分思维-动作对) 进行匹配。

若局部动作与“标准答案”高度相似，给予部分奖励。

混合计算组

Gcalc = Gonline ∪ Gbuffer

当在线探索 ($\mathcal{G}_{\text{online}}$) 全军覆没时，STDB 提供的锚点能够维持梯度更新，确保模型持续学习，避免“零梯度”陷阱。
